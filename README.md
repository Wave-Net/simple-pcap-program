# simple-pcap-program
다양한 라이브러리로 간단한 패킷 캡쳐 프로그램 만들어보기

## Commit Message Rule
| 타입    | 설명                                       |
|---------|--------------------------------------------|
| ``feat``    | 새로운 기능 추가, 기존의 기능을 요구 사항에 맞추어 수정 |
| ``fix``     | 버그 수정                                    |
| ``refactor`` | 기능의 변화가 아닌 코드 리팩터링 ex) 변수 이름 변경     |
| ``docs``    | 문서(주석) 수정                                |
| ``style``   | 코드 formatting, 세미콜론(;) 누락, 코드 변경이 없는 경우 |
| ``test``    | 테스트 코드, 리팩토링 테스트 코드 추가         |
| ``chore``   | 기타 수정 ex) .gitignore             |
